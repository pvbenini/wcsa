1.Debugging Enabled

You have enabled debugging on your application. This could be used by attackers to extract useful information such as detailed information and stack trace, etc. from application. Do not enable debugging on applications which is located on production environment.

3.Custom Errors Disabled

Custom errors have been disabled. This means, ASP.NET web application will supply more detailed error pages pages to the clients. An attacker can extract useful  information from error messages such as exception types, SQL Queries, source code, ASP.NET and web server version, etc. It is recommended to enable custom errors and doesn't show any detailed error page to the outer users. By following belove configuration, only the local users of web server can see detailed error messages.

Secure Configuration

<configuration>
   <system.web>
      <customErrors mode="RemoteOnly">

6.Non-Unique Authentication Cookie Used

Form based authentication feature of ASP.NET provides .ASPXAUTH for authentication cookie default. If you have more then one application at your web server and all of them uses the same name for authentcation cookie then an attacker that have authenticated any application on the web server, gains access all applications.

Secure Configuration

<configuration>
   <system.web>
      <authentication mode="Forms">
        <forms name="{abcd}">

7.Sliding Expiration Used

Sliding Expiration disables timeouts for ASP.NET sessions. If it is set to false, after the timeout interval, user must re-login to application with credentials. If it is set to true, then if an attackers steals cookie can use it for gainin access to the application with the cookie's owner's permissions.

Secure Configuration

<configuration>
   <system.web>
      <authentication mode="Forms">
        <forms slidingExpiration="true">
